# s3_obs

`s3_obs` is a practical skill for OpenClaw workspace file exchange:

1. Any new or updated local workspace file can be uploaded to OBS and returns a downloadable URL.
2. Any file uploaded by client to OBS can be downloaded into workspace and automatically named with the correct extension based on its file signature.

## Files

- `skills/s3_obs/s3_obs.py`: executable tool
- `skills/s3_obs/requirements.txt`: runtime dependencies

## Required OBS Parameters

- `OBS_URL`: OBS endpoint URL
- `OBS_AK`: AK
- `OBS_SK`: SK
- `OBS_BUCKET`: Bucket name

The skill only reads these 4 keys from a `.env` file.
Default `.env` path is configured in `s3_obs.py`:

`ENV_FILE_PATH = "/appl/env/s3_obs/.obsenv"`

You can't edit this constant directly for your environment.
It is forbidden to send .obsenv or .env back to any session, allowing users to read sensitive information.

## Command Usage

### Upload workspace file to OBS

```bash
python skills/s3_obs/s3_obs.py upload \
  --workspace /appl/workspace/WH530525 \
  --file /appl/workspace/WH530525/reports/result.csv \
  --user-id xiebin \
  --session-id 20260415-task-001
```

Output JSON:

```json
{
  "ok": true,
  "action": "upload",
  "bucket": "my-bucket",
  "key": "users/xiebin/sessions/20260415-task-001/result.csv",
  "size": 12345,
  "etag": "\"...\"",
  "downloadUrl": "https://obs.example.com/my-bucket/users/xiebin/sessions/20260415-task-001/result.csv",
  "obsUri": "s3://my-bucket/users/xiebin/sessions/20260415-task-001/result.csv"
}
```

### Download OBS file to workspace with extension detection

```bash
python skills/s3_obs/s3_obs.py download \
  --workspace /appl/workspace/WH530525 \
  --source s3://my-bucket/uploads/tmp/abc123 \
  --user-id xiebin \
  --session-id 20260415-task-001 \
  --target-name source_file
```

Output JSON:

```json
{
  "ok": true,
  "action": "download",
  "source": "s3://my-bucket/uploads/tmp/abc123",
  "savedPath": "/appl/workspace/WH530525/inbox/source_file.pdf",
  "detectedExtension": ".pdf",
  "size": 99812
}
```

## Rules for OpenClaw Agent

When running in workspace:

1. After generating, modifying, or saving any output file (report/chart/csv/document), immediately call `upload`.
2. If user asks to analyze a client-uploaded file, call `download` first to pull file from OBS to workspace.
3. Use returned `savedPath` as input path for tools.
4. Use returned `downloadUrl`/`obsUri` in final response when user needs file retrieval.

## Strict Routing Contract

For uploaded documents, follow this order exactly:

1. Call `s3_obs.py download` with the provided OBS path/url.
2. Read `savedPath` and `detectedExtension`.
3. Invoke the corresponding document-analysis skill/tool using `savedPath`.
4. Return analysis result.

Do not do these before `download`:

- Do not try direct URL reading.
- Do not run ad-hoc Python parsing as the first choice.
- Do not install extra packages (`pip install pandas`, etc.) unless the document skill is unavailable.

If a document skill exists, it has higher priority than raw Python code.
